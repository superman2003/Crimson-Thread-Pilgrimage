# Godot生成地图能力顺序与软锁验证报告

时间：2026-06-19 14:37:01

## 结果

本轮为 generated Godot 地图增加能力获取顺序验证：验证器会在不穿过未解锁 `ability_gate` 的前提下，从出生点逐步寻找可达能力物品、解锁对应门，再确认 Boss、出口和全图房间最终可达。

## 改动

- `tools/validate_generated_godot_runtime.py`
  - 新增 `reachable_nodes()`，用于从出生点计算当前可达房间集合。
  - 新增 `progression_graph(config, abilities)`，根据已获得能力决定哪些 `ability_gate` 可通行。
  - 新增 `ability_collectible_rooms(config)`，把能力物品定位到房间。
  - 新增 `validate_ability_unlock_order(config)`，模拟“探索 -> 拿能力 -> 开门 -> 继续探索”的类银河城进度链。
  - 输出每章 `ability_order` 和 `reachable_rooms_after_unlocks`，用于快速检查是否存在软锁。

## 风险

- 当前图算法把 `shortcut` 视为拓扑可用连接；后续如果要模拟“背面开启前单向不可用”，需要给 `shortcut` 增加 `opened_by_room` 或 `initially_open` 字段。
- 当前能力顺序验证是房间拓扑级，不是逐像素/逐平台路径搜索；平台物理可达性仍由基础连接几何验证覆盖。
- Godot headless 日志仍包含既有插件和导入缓存噪声，但验证脚本返回成功。

## 下一步

1. 给 `shortcut` 增加“背面开启”方向语义，验证初始路线不依赖未开启捷径。
2. 做游戏内 debug overlay，显示能力链、剩余能力门和可达区域。
3. 把能力门视觉升级为门框、符号和开门反馈，而不是纯静态阻挡体。
