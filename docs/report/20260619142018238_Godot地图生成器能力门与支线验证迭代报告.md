# Godot地图生成器能力门与支线验证迭代报告

时间：2026-06-19 14:20:18

## 结果

本轮把 `blueprint -> Godot demo JSON` 生成器从“拓扑注释”推进到“可执行进度契约”：生成 JSON 现在显式包含主线、支线、捷径、能力门、锁门体和调试地图摘要，并通过 Python 与 Godot headless 验证。

## 改动

- `tools/blueprint_to_godot_runtime.py`
  - 为连接增加 `id`、`progression_tier`、`traversal`、`directionality`、`requires_unlock`。
  - 为能力门增加 `gate_id`、`required_ability`、`required_ability_name`、`locked`。
  - 生成 `locked_gates`，供 Godot 运行时创建能力门阻挡体。
  - 生成 `debug_map`，记录主线、支线、捷径、能力门和房间中心点。
  - 支线连接也生成可达支撑平台，减少“地图连通但玩家跳不过去”的假阳性。
  - 大高差连接改为阶梯链，避免支线/主线垂直差过大导致不可达。
- `tools/validate_generated_godot_runtime.py`
  - 主路几何验证升级为基础移动连接验证，覆盖主线和支线。
  - 新增能力门、锁门体、能力拾取物、捷径拉杆、调试地图一致性验证。
  - 平台密度阈值调整到 `8.5`，匹配主线+支线均生成支撑平台后的灰盒体量。
- `tools/render_generated_godot_runtime_preview.py`
  - 预览图绘制连接线：主线、支线、捷径、能力门分色显示。
- `godot/scripts/generated_runtime_validation.gd`
  - Godot headless 增加连接进度契约、能力门锁体、debug map 断言。
- `godot/data/generated/*.generated.json`
  - 重新生成 6 章 Godot demo JSON。
- `artifacts/map_blueprints/godot_generated_preview/*`
  - 重新生成 6 章预览图和总览 contact sheet。

## 风险

- Godot headless 仍会打印 MetroidvaniaSystem 插件 `MapView` 解析错误，但生成地图校验脚本本身返回成功。
- 当前 `locked_gates` 已生成静态阻挡体和能力需求字段，但完整“拾取能力后自动开指定门”的运行时交互还需要下一轮接入。
- 生成器仍是灰盒可玩地图，不是最终美术 TileMap；后续需要把房间模板映射到真实瓦片和环境资产。

## 下一步

1. 在 Godot 运行时接入能力拾取与 `locked_gates` 解锁逻辑。
2. 增加生成地图 debug overlay，可在游戏内显示主线/支线/捷径/能力门。
3. 把 JSON 灰盒平台逐步映射到主题化 TileMap 或场景块。
