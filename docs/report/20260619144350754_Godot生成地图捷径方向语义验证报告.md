# Godot生成地图捷径方向语义验证报告

时间：2026-06-19 14:43:50

## 结果

本轮为 generated Godot 地图补上 `shortcut` 的背面开启方向语义。生成 JSON 现在能区分“初始只能从背面到正面”和“打开后双向通行”，验证器也会按初始单向规则重新计算能力顺序。

## 改动

- `tools/blueprint_to_godot_runtime.py`
  - `shortcut` 连接新增：
    - `opens_from_room`
    - `return_to_room`
    - `initial_directionality: from_to_only`
    - `opened_directionality: two_way`
  - shortcut lever 的 `opens_connection` 同步写入方向字段。
- `tools/validate_generated_godot_runtime.py`
  - shortcut 静态校验要求具备方向字段。
  - `progression_graph()` 新增 `shortcuts_open` 参数。
  - 能力顺序验证阶段按 `shortcuts_open=False` 计算，避免初始路线依赖未开启捷径反向边。
  - 全图最终可达阶段按 `shortcuts_open=True` 计算，模拟所有捷径已打开后的回环。
- `godot/scripts/generated_runtime_validation.gd`
  - 校验 shortcut 连接和 lever 都具备方向契约。

## 验证

- `npm run build:generated-maps`：通过。
- `npm run test:generated-maps`：通过。
- `D:\Godot\godot.cmd --headless --path godot --script res://scripts/generated_runtime_validation.gd`：通过，关键输出在：
  - `artifacts/map_blueprints/godot_generated_preview/generated_runtime_validation_latest.log`

## 观察

当前 6 章在 `shortcuts_open=False` 时也能全图可达，说明现有 shortcut 主要是“回环/回跑优化”标注，不是主线必需路径。能力顺序仍稳定通过，所有章节最终可达房间数为满额。

## 风险

- 现有生成器还没有强制“捷径缩短 Boss runback 或连接远端 hub”的价值评分。
- Godot 运行时目前保存 lever 的方向契约，但实际 gameplay 仍使用统一 `shortcut_open` 状态，后续若要多捷径逐个开启，需要扩展为 `opened_shortcuts` 字典。

## 下一步

1. 给 shortcut 增加价值评分：打开后必须缩短 boss/save/hub 间距离。
2. 将运行时 `shortcut_open` 升级为多捷径 `opened_shortcuts`。
3. 在游戏内 debug overlay 显示已开启/未开启捷径和能力门状态。
