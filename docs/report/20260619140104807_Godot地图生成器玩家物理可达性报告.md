# Godot 地图生成器玩家物理可达性报告

## 结果

generated 地图验证已从拓扑连通升级到玩家物理可达性检查。验证器会读取 Godot 玩家控制器参数，计算跳跃高度、空中水平距离和 dash 距离，并检查主路连接是否真的能跳过去。

## 玩家参数来源

文件：`godot/scripts/player_controller.gd`

- `speed = 285.0`
- `jump_velocity = -710.0`
- `gravity = 1850.0`
- `dash_speed = 780.0`
- `dash_time = 0.16`

当前验证计算：

- 跳跃高度约 `136.2px`
- 跳跃 + dash 保守水平可达约 `281.7px`

## 本轮修复

- `tools/validate_generated_godot_runtime.py`
  - 新增玩家物理参数解析。
  - 新增平台之间可达性判定。
  - 主路连接现在会检查地面、桥、阶梯、中继平台组成的链路是否可跳。
- `tools/blueprint_to_godot_runtime.py`
  - 修复长距离区域连接，只生成短中继平台，不再只放一块短桥导致主路断裂。
  - 保持灰盒节奏为可跳平台链，而不是整段实心桥。

## 发现的问题

物理验证首次运行抓到 CH01 主路断点：

- `ch01_07 -> ch01_08`
- gap `1047px`
- 高差 `157px`
- 玩家保守可达 `281.7px`

修复后该类长 gap 会自动生成分段中继平台。

## 验证

- `npm run test:generated-maps`
- `node tools/run_python_tool.mjs tools/validate_silksong_scale_map_blueprints.py`

全部通过。Godot 仍会输出 MetroidvaniaSystem 插件 `MapView` 解析错误，但 generated runtime 验证脚本成功返回。
