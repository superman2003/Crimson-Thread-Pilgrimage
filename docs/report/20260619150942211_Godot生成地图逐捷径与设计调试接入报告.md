# Godot 生成地图逐捷径与设计调试接入报告

时间：2026-06-19 15:09:42

## 结果

已把生成地图从“全局一个 shortcut_open”升级为“逐捷径状态”，并把房间设计契约接入 Godot 地图面板和 debug state。

这一步让 `blueprint -> Godot demo JSON` 更接近可策划地图工具：每条捷径可以独立追踪，当前房间的 archetype、节奏职责、移动玩法和奖励契约也能在运行时读出来。

## 改动

| 文件 | 改动 |
| --- | --- |
| `godot/scripts/main_level.gd` | 新增 `opened_shortcuts` 字典、逐捷径 helper、`_debug_open_all_shortcuts()`、地图面板设计契约显示、debug state 输出 |
| `godot/scripts/generated_runtime_validation.gd` | Godot headless 验证逐捷径总数、初始关闭状态、debug 全开后状态，以及当前房间 archetype |
| `tools/validate_generated_godot_runtime.py` | 静态验证 `main_level.gd` 必须包含逐捷径和房间设计 debug 契约 |

## 运行时状态

新增或强化的 debug 字段：

- `shortcut_total`
- `opened_shortcut_total`
- `opened_shortcut_ids`
- `current_room_archetype`
- `current_room_pacing_role`
- `current_room_traversal_focus`
- `current_room_reward_contract`

保留兼容字段：

- `shortcut_open`

`shortcut_open` 现在表示至少有一条捷径或 Boss 路线开启；生成地图真正的多捷径状态以 `opened_shortcuts` 为准。

## 地图面板新增信息

地图 overlay 现在会显示：

- 当前房间 archetype
- 当前房间 pacing role
- traversal focus
- reward contract
- 已开启捷径数 / 总捷径数

这让后续调 CH01 纵切时能直接在游戏内检查房间是否承担正确职责。
