# 类丝之歌地图生成器自然簇迭代报告

## 结果

已把地图生成从固定三条 lane 的规则排布，推进到区域簇形态模板驱动。蓝图和 Godot generated JSON 都已重建，并通过蓝图、Godot JSON、Godot headless 三层验证。

## 本轮改动

- `tools/build_silksong_scale_map_blueprints.py`
  - 新增区域簇形态：入口脊线、纵向井、环形走廊、奖励洞、横向渠、Boss 前短跑。
  - 房间坐标改为非等距自然分布，不再按固定列和三条水平线排列。
- `tools/blueprint_to_godot_runtime.py`
  - Godot 物理布局改为从蓝图 `rect.x/y/w/h` 映射。
  - 收紧连接支撑平台，短 gap 不再强行铺整桥，减少平台噪声。
- `tools/validate_silksong_scale_map_blueprints.py`
  - 新增布局自然度检查：X/Y 坐标桶数量、宽簇/高簇数量。

## 当前数据

- 蓝图：六章共 64 / 68 / 66 / 72 / 70 / 74 房。
- Godot generated：六章平台数降到 287 / 300 / 294 / 320 / 305 / 329。
- 敌人维持探索友好密度：33 / 41 / 37 / 40 / 41 / 38。
- Godot 世界宽度仍控制在 20930-22431，未退回超长横向串联。

## 仍需注意

- 这是灰盒可玩地图，不是最终 TileMap 美术。
- Godot headless 启动仍会打印 MetroidvaniaSystem 插件 `MapView` 解析错误，但 generated runtime 验证脚本成功返回。
- 下一轮应加入玩家跳跃/冲刺能力模拟，检查平台跨度不是只在拓扑上连通，而是手感上可达。
