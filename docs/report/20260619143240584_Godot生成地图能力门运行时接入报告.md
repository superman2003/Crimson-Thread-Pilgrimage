# Godot生成地图能力门运行时接入报告

时间：2026-06-19 14:32:40

## 结果

本轮把生成地图的能力门从静态 JSON 契约接入到 Godot 运行时：生成地图在 `combat_clear` 模式下仍会刷出能力物品，拾取能力后会按 `locked_gates.required_ability` 解锁对应门体。

## 改动

- `godot/scripts/main_level.gd`
  - 新增 `locked_gate_bodies`，按 `gate_id -> StaticBody2D` 管理多能力门，避免旧的单 `gate_body` 覆盖问题。
  - 新增 `unlocked_abilities`，记录已获得能力。
  - `combat_clear` 模式下只生成 `kind == "item"` 的能力收藏物，避免把普通货币奖励全部塞进战斗推进节奏。
  - `_collectible()` 写入 `item_id`、`amount`、`ability_name`。
  - `_on_collectible_body_entered()` 区分 `currency` 和 `item`：货币进货币计数，能力物品进背包并刷新能力门。
  - 新增 `_refresh_ability_gates()` 与 `_unlock_ability_gate()`。
  - `get_demo_state()` 输出 `unlocked_ability_total`、`unlocked_ability_ids`、`locked_gate_total`、`locked_gate_remaining`。
  - 新增 `_debug_collect_all_abilities()` 供 headless 验证调用。
- `godot/scripts/generated_runtime_validation.gd`
  - 静态校验增强：能力门需求必须能在 `collectibles[].item_id` 找到。
  - 运行时校验增强：实例化 CH01 generated 场景，确认初始能力门全部锁住，调用 `_debug_collect_all_abilities()` 后剩余能力门为 0。
- `tools/validate_generated_godot_runtime.py`
  - 新增对 `main_level.gd` 能力门运行时契约的静态检查。

## 验证

- `npm run test:generated-maps`：通过。
- `D:\Godot\godot.cmd --headless --path godot --script res://scripts/generated_runtime_validation.gd`：通过，输出 `GENERATED_RUNTIME_VALIDATION_PASS configs=6`。

## 风险

- Godot headless 仍会打印现有 MetroidvaniaSystem 插件 `MapView` 解析错误和部分导入纹理缓存警告；本次生成地图验证脚本返回成功。
- 运行时解锁验证目前只实例化 CH01 做动态验证，其余 5 章做静态契约验证；这样能避免 6 章完整场景实例化导致 headless 超时。
- 真实玩家流程还没有做“按地图路线拿到能力再自然到门口”的全流程游玩验证，下一步需要加 debug overlay 或自动路径验证。
