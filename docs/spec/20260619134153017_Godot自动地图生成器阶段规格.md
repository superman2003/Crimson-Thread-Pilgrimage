# Godot 自动地图生成器阶段规格

## 目标

自研 `blueprint -> Godot demo JSON` 生成器，用通用银河城结构原则生成可直接加载的横版探索地图。参考方向是大体量、多支线、能力门、捷径闭环、Boss 前短跑；不复刻任何商业地图。

## 当前闭环

- 蓝图来源：`artifacts/map_blueprints/silksong_scale/*_blueprint.json`
- 生成器：`tools/blueprint_to_godot_runtime.py`
- 输出：`godot/data/generated/*.generated.json`
- 入口场景：`godot/scenes/GeneratedMain.tscn`
- 预览图：`artifacts/map_blueprints/godot_generated_preview/generated_six_chapter_preview_contact_sheet.png`
- 静态验证：`npm run test:generated-maps`
- Godot 验证：`D:\Godot\godot.cmd --headless --path godot --script res://scripts/generated_runtime_validation.gd`

## 已实现规则

- 每章 55+ 房间，六章 generated JSON 全覆盖。
- 逻辑房间 range 与实际物理 `layout_rect/visit_rects` 分离。
- 按房间职责生成地貌模板：入口、Hub、主路、上层奖励口袋、下层回访洞、Boss arena。
- 主路、局部分支自动补物理桥和阶梯；捷径和能力门保留为连接语义与交互数据。
- 敌人密度从每章约 95-108 降到约 32-42，避免探索节奏被战斗淹没。
- 章节出口链路已串为 CH01 generated -> CH06 generated。

## 验证门槛

- 六章 generated JSON 均存在并可解析。
- 房间、平台、敌人、危险区、NPC、存档点、交互和连接字段齐全。
- 出生点能连到 Boss 和出口。
- Boss 前 3 步内有存档点。
- 关键点位在房间 `layout_rect/visit_rects` 内。
- 平台和敌人密度有上限，防止退化成平台森林。
- Godot headless 脚本验证六章 generated 配置和章节链路。

## 当前风险

- 预览仍偏规则化，蓝图房间分布本身还需要更自然的区域簇形态。
- Godot 启动会输出 MetroidvaniaSystem 插件 `MapView` 解析错误，但 generated runtime 验证脚本已完成并返回 0。
- 当前是灰盒可玩数据，不是最终美术 TileMap；后续需要把地貌模板升级成更明确的区域玩法语言。

## 下一轮方向

1. 让蓝图生成阶段支持区域簇模板：横向廊、竖井、环道、奖励洞、Boss 前短路。
2. 增加可达性模拟：按玩家跳跃、冲刺和能力门验证平台跨度。
3. 让 Godot 预览或调试 HUD 展示主线、支线、捷径、能力门和回访路径。
