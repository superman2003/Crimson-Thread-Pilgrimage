# 类丝之歌地图自动生成器规格

## 目标

- 用代码把 `artifacts/map_blueprints/silksong_scale/*_blueprint.json` 转成 Godot 可直接读取的运行时 JSON。
- 不复刻任何商业地图，只借鉴大体量银河恶魔城的通用结构：主线、支线、能力门、捷径闭环、Boss 前短跑。
- 第一阶段输出到 `godot/data/generated/`，不覆盖现有手工章节数据。

## 已交付原型

- 生成器：`tools/blueprint_to_godot_runtime.py`
- Python 验证器：`tools/validate_generated_godot_runtime.py`
- Godot 验证脚本：`godot/scripts/generated_runtime_validation.gd`
- 已生成：
  - `godot/data/generated/demo_ch01_moss_bell_court.generated.json`
  - `godot/data/generated/demo_ch02_rain_foundry_canal.generated.json`

  - `godot/data/generated/demo_ch06_silent_crown_core.generated.json`

## 生成规则

1. 每章 55-75 个房间、8-12 个区域簇；主线路径占总房间 50%-62%。
2. 每个区域簇 5-8 个房间，至少包含入口/过渡房、主线推进房、支线或奖励房。
3. 普通房以 1-2 屏宽为主；枢纽、纵井、Boss 房可放大到 2-4 屏。
4. 主线形成“长推进 + 局部回环”，每 6-9 个主线房出现一次存档、捷径、能力点、NPC、精英战或区域切换。
5. 每个区域簇至少 1-3 条支线；超过 4 房的支线必须提供奖励、捷径或回访门。
6. 每章至少 5-6 个能力门，其中 40%-60% 放在首次经过但暂不能进入的位置。
7. 能力点之后 1-3 房内必须有教学验证房，3-6 房内必须有正式能力门或捷径收益。
8. 每章至少 8 条捷径；捷径从后段或高风险路径反向打开，把跑图距离缩短到原路线的 30%-60%。
9. Boss 前必须有存档点或等价检查点；从存档到 Boss 门不超过 2-3 个房间。
10. 奖励死胡同长度控制在 1-3 房；末端必须有奖励、隐藏剧情、资源补给或能力挑战。
11. 每章至少 1-2 条垂直井或纵向主干，连接上下层 3 个以上关键节点。
12. 获得新能力后，应解锁本章 2-4 个旧区域入口，并至少连到 1 个主线捷径、1 个奖励死胡同、1 个新区域入口或剧情节点。
13. 每章至少 5 个存档点，推荐 6-7 个；入口、中段枢纽、能力后、Boss 前都要覆盖。
14. 风险曲线按入口低压、中段学习、能力验证、Boss 前压强递进。
15. 生成器必须校验连通性、主线可达性、能力门、捷径、Boss 前短跑、奖励死路非空、每簇体量和存档覆盖。

## Godot JSON 输出字段

生成器保留原章节故事、素材、Boss、AI 配置，并替换或生成以下字段：

- `id/name/map_title/art_theme/progression_mode`
- `world/fall_y/player_start/boss_checkpoint`
- `map_rooms`
- `platforms`
- `connections`
- `save_points`
- `npcs`
- `collectibles`
- `hazards`
- `enemy_spawns`
- `interactives`
- `chapter_transition`
- `entry_from_previous`
- `generation`

验证覆盖：

- 必填字段存在。
- 房间 `range` 合法且不重叠。
- 连接引用有效，出生点可 BFS 到 Boss 与出口。
- 平台密度大于 `rooms * 2`。
- 重要点位落在房间 `visit_rects/layout_rect/range` 内。
- 敌人 `platform_id` 引用有效，`kind` 存在于 `ai_profiles`。
- Boss 前 3 步内有存档点。
- 六章生成预览图：`artifacts/map_blueprints/godot_generated_preview/generated_six_chapter_preview_contact_sheet.png`

## 风险

- 当前灰盒是“能跑通”的结构验证版，尚未做逐跳跃物理精校。
- 地图非常长，后续需要按区域流式加载或分场景优化。
- 敌人数量按房间自动铺设，第二阶段需要按强度曲线减少或分批激活。
- 目前只生成 CH01/CH02；六章批量生成前应先用 Godot CLI 实机加载验证。
