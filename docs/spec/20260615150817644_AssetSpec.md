## 3. 命名规范

### 3.1 文件名格式

```text
{category}_{scope}_{subject}_{variant}_v{nnn}.png
```

- `category`：`chr`、`wpn`、`enm`、`boss`、`env`、`map`、`tile`、`vfx`、`ui`
- `scope`：`player`、`global`、`ch01` ~ `ch06`
- `subject`：实体或用途 slug，例如 `lanra`、`rust-crown-guard`、`moss-forge`
- `variant`：动作/状态/用途，例如 `idle-sheet`、`arena-bg`、`minimap`
- `v{nnn}`：版本号，今天统一从 `v001` 开始

### 3.2 约束

- 只允许小写英文字母、数字、短横线、下划线和 `.png`。
- 禁止空格、中文、括号、`final`、`new`、`copy`。
- 同一 asset_id 不覆盖旧文件，后续重生成必须递增版本：`v002`、`v003`。
- 临时占位素材必须带 `_placeholder`，正式批量验收不允许出现 `_placeholder`。

## 4. PNG 尺寸标准

| 类别 | 用途 | PNG 尺寸 | 帧约定 | 背景 |
|---|---:|---:|---:|---|
| 角色动画 sheet | 主角动作 | `1024x128` | 8 帧，每帧 `128x128` | 透明 |
| 角色肖像 | 对话/菜单 | `512x512` | 单图 | 透明 |
| 武器/攻击轨迹 sheet | 刃光、钩索、反闪、治疗 | `1024x128` | 8 帧，每帧 `128x128` | 透明 |
| 普通怪物 sheet | crawler/flyer/shooter/charger | `768x96` | 8 帧，每帧 `96x96` | 透明 |
| Boss sheet | Boss 核心动作 | `1536x192` | 8 帧，每帧 `192x192` | 透明 |
| 章节远景背景 | 章节环境 | `1920x1080` | 单图，可横向裁切 | 不透明 |
| 地图预览 | 当前关卡预览 | `3260x540` | 单图，匹配 `SETTINGS.levelWidth`/`height` | 不透明 |
| tileset | 地形、平台、机关、危险物 | `1024x1024` | `32x32` tile 网格 | 透明 |
| VFX sheet | 战斗/章节/Boss 特效 | `1024x128` | 8 帧，每帧 `128x128` | 透明 |
| UI 图标 | HUD/menu/icon | `256x256` | 单图 | 透明 |
| UI 条/面板 | 血条、能量条、菜单面板 | `1024x256` | 单图或 4 段九宫格 | 透明 |

## 8. Manifest 字段

`asset_manifest.json` 顶层建议结构：

```json
{
  "project": "crimson-thread-odyssey",
  "display_name_zh": "绯线远征",
  "asset_root": "D:/CrimsonThreadOdysseyAssets",
  "generated_date": "2026-06-15",
  "version": "2026.06.15.asset-batch.v001",
  "schema_version": 1,
  "minimum_total_png": 110,
  "assets": []
}
```

单个 asset 必填字段：

| 字段 | 类型 | 必填 | 说明 |

| `category` | string | 是 | `character`/`weapon`/`enemy`/`boss`/`environment`/`map`/`tile`/`vfx`/`ui` |

| `chapter_no` | number/null | 是 | 全局资产为 `null`，章节资产为 `1`~`6` |
| `chapter_id` | string/null | 是 | 例如 `moss-forge` |
| `chapter_dir` | string/null | 是 | 例如 `ch01_moss-forge` |

| `source_entity` | string/null | 是 | 对应角色/怪物/Boss/机制名 |
| `file_name` | string | 是 | PNG 文件名 |

| `width` | number | 是 | PNG 宽 |
| `height` | number | 是 | PNG 高 |
| `frame_width` | number/null | 是 | 非 sheet 为 `null` |
| `frame_height` | number/null | 是 | 非 sheet 为 `null` |
| `frames` | number | 是 | 单图为 `1` |
| `columns` | number | 是 | 单图为 `1`，sheet 为 `8` |
| `rows` | number | 是 | 今天统一 `1`，tileset 可写 `32` |
| `transparent_bg` | boolean | 是 | 远景/地图一般为 `false`，其他多为 `true` |
| `palette` | string[] | 是 | 主要色值数组，至少 3 个 |
| `usage_tags` | string[] | 是 | 例如 `["runtime","player","combat"]` |
| `prompt` | string | 是 | 生成时使用的简短提示词 |
| `license` | string | 是 | 固定 `original-generated` |

## 10. 验收标准

- `D:\CrimsonThreadOdysseyAssets\asset_manifest.json` 存在且 JSON 可解析。
- manifest `assets.length >= 110`。
- 每个 manifest 条目对应 PNG 文件存在。
- 每个 PNG 头部合法，尺寸与 manifest 完全一致。
- 每类最低数量满足第 5 节。
- 文件名只含 ASCII 安全字符，不出现 `_placeholder`、`copy`、`final`、`new`。
- 验收报告生成到：`D:\CrimsonThreadOdysseyAssets\validation\reports\asset-validation-report.json`

## 11. 风险与回滚

- 风险：一次生成 110 张图可能风格漂移；回滚策略是保留 manifest 中 `status=rejected` 的条目，重生成同 `asset_id` 的 `v002` 文件，不删除 `v001`。
- 风险：AI 生成 sprite sheet 帧边界可能不齐；回滚策略是先验收尺寸，再由后续切帧脚本检查每帧透明边距和主体居中。
- 风险：中文路径在部分工具中乱码；本规范将 D 盘最终资产目录与文件名全 ASCII，中文只在 manifest 字段中出现。
- 风险：当前游戏仍以 Canvas 绘制为主；素材批量生成后需要独立 AssetRegistry 接入，不在本规格范围内。
