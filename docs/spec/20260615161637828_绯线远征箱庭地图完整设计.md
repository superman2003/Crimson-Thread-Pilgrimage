## 0. 设计目标

本作地图不做单向横版关卡，而做“章节内小型箱庭”。每章都要给玩家一次“柳暗花明又一村”的体验：

1. 入口 1-2 分钟内看见目标：Boss 场、出口、宝箱、灯碑或锁门背面。
2. 正面被阻断：铁门、水闸、断桥、热浪、虚实平台、核心封印。
3. 绕路学习机制：从外环、下层、镜中、树洞、水渠或回声室绕到背面。
4. 从另一侧打开捷径：单向门、电梯、水闸、落桥、环门、树洞升降机。
5. 回到熟悉地点：玩家意识到“原来刚才看见的是这里”，产生空间理解。

## 1. 通用节点标记

| 标记 | 含义 | 设计要求 |
|---|---|---|
| `S` | 章节入口 | 安全进入，展示本章主视觉 |
| `BF` | 灯碑/复活点 | Boss 前必须有一个 |
| `V` | 远景预告点 | 看见目标但暂时到不了 |
| `D` | 正面阻断门 | 必须从背面、机关或能力打开 |
| `N` | 主线房间 | 机制教学、战斗、平台挑战 |
| `SH` | 安全屋 | 观察机制、降压、NPC/文本 |
| `G` | 能力门 | 需要钩索、二段跳、斜冲、反闪等 |
| `SC` | 捷径 | 从深处回连早期区域 |
| `Sx` | 支线 | 奖励、素材、护符、剧情碎片 |
| `E` | 精英战 | Boss 前机制考试 |
| `B` | Boss 场 | 复用本章核心机制 |
| `X` | 出口 | 通往下一章或返场中枢 |

## 9. 数据落地建议

配套文件：

- `docs/spec/20260615161637828_soulbox_level_graph.json`
- `docs/spec/20260615161637828_soulbox_level_graph.mmd`

Godot 灰盒建议：

1. 每章先实现 `BF1 -> 主线 -> SC1 -> BF2 -> Boss` 最小闭环。
2. `nodes` 表示房间，`edges` 表示连通，`locks` 表示能力门，`shortcuts` 表示从背面开启的边。
3. 每个陷阱必须落 `telegraph`，禁止无预警秒杀。
4. Boss 前复跑路控制在 8-10 秒内。
5. 第 6 章 `HUB` 必须做成真正中枢，而不是线性五连房。
