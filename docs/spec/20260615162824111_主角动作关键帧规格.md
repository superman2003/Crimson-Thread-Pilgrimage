## 1. 统一规格

### 1.1 帧画布与轴心

- 建议单帧画布：`128x128`，透明背景。
- 角色视觉轴心：脚底中心，建议 Godot 本地原点为 `(0, 0)`。
- Sprite 帧内脚底中心建议落在像素坐标 `(64, 104)`，头部和灯芯结构保留上方余量。
- 朝向默认向右；向左移动和攻击用 Godot 水平翻转，hitbox 的 x 坐标同步取反。
- 动作位移优先由 `CharacterBody2D` 物理代码驱动，动画文件只保留微弱视觉跟随，不直接烘焙大段 root motion。

### 1.2 Godot 盒体约定

盒体使用 Godot 本地坐标，角色脚底中心为 `(0, 0)`，x 向右，y 向下。因此头部和躯干中心 y 为负值。

- 主碰撞体：`CollisionShape2D`，建议矩形或胶囊，默认 `size=(24,48) center=(0,-24)`。
- 受击盒：`Area2D/Hurtbox`，默认 `size=(28,50) center=(0,-26)`，比主碰撞体略宽，用于容错。
- 命中盒：`Area2D/Hitbox`，仅攻击帧启用，表中写明 active frames。
- 地面检测：不写入本规格，但建议 A0 灰盒保留脚底 `RayCast2D` 或 shape cast。
- 帧编号从 1 开始，例如 `active 3-4` 表示第 3 和第 4 帧启用命中盒。

## 3. 动作节奏和状态切换

### 3.1 地面移动

- `idle -> run`：检测水平输入立即切换，run 第 1 帧可直接播放，不需要额外起步动画。
- `run -> idle`：速度低于阈值时切回 idle；若高速刹车，允许复用 `land` 的 1-3 帧作为短刹车。
- `run -> jump_start`：输入 jump 后播放 `jump_start`，第 3 或第 4 帧实际给起跳速度。
- `fall -> land`：落地速度较低时可直接 idle/run；中高落速播放完整 `land`。

### 3.2 空中动作

- `jump_start -> jump_loop`：垂直速度仍向上时进入。
- `jump_loop -> fall`：垂直速度大于等于 0 时进入。
- `fall -> wall_slide`：贴墙、按墙方向、垂直速度向下时进入。
- `air_attack` 播放期间不改变主 hurtbox，hitbox 只在 3-5 帧启用。

### 3.3 冲刺和攻击

- `dash` 优先级高于普通移动，低于 hurt/death。
- `dash` 1-4 帧可做无敌窗口，若设计不需要无敌，则保留半高 hurtbox。
- `attack_1 -> attack_2`：建议在 `attack_1` 第 4-6 帧开放连段输入缓存。
- `attack_2` 第 7-8 帧开放 dash/jump 取消，手感更利落。
- `hook_throw` 第 4 帧发射钩线；命中、拉拽、断线逻辑交给钩索系统。

## 5. 美术绘制要点

- Lumen 的识别点是玻璃灯芯头部、黄铜线轮、青绿色发光线和工匠短外套；不要使用白面具、虫角、红披风、细针剑等旧主角高风险元素。
- `idle/run/jump/wall_slide/attack` 可从现有 `lumen_01` 到 `lumen_05` 单图提取姿态语言，但需要重画为完整序列。
- 每个动作要保持脚底轴心稳定，尤其 idle/run/land，避免 Godot 播放时角色上下跳。
- 攻击动作的环刃拖尾可以做在角色 sheet 内，也可以拆成独立 VFX；若拆 VFX，角色 hitbox 时间仍以本规格为准。
- dash 的残影不要扩大实际 hurtbox；视觉可以长，判定要短。

## 6. A0/Godot 接入验收清单

- `SpriteFrames` 内存在 14 个动画名，且名称与第 1.3 节一致。
- 每个动画帧数、FPS、循环设置与第 2 节一致。
- idle/run/jump/fall/land/dash/wall_slide/hurt/death 不产生攻击 hitbox，除 dash 破物玩法明确开启。
- attack_1、attack_2、air_attack、hook_throw 的 active frames 与第 2 节一致。
- 角色向左时，视觉翻转和 hitbox x 坐标镜像一致。
- Godot 主碰撞体不随美术拖尾变大，拖尾只做视觉和 VFX。
- 死亡第 6 帧后 hurtbox 关闭，避免倒地后继续受击。
