# Godot引擎对标优化规格

## 目标

- 交付目标以 Godot 工程为准，不再把网页版作为主目标。
- 工程入口：`godot/project.godot`
- 主场景：`res://scenes/Main.tscn`
- 本地验证引擎：Godot `4.6.3.stable`

## 对标边界

- 参考方向：空洞骑士 / 丝之歌式 2D 横版探索、清晰前景平台、近战读招、钥匙-捷径-Boss 的小闭环。
- 不复刻商业素材、角色、地图、美术设定；只对标手感、结构、可读性和关卡组织方式。
- 对标资料源：
  - Hollow Knight 官方站：https://www.hollowknight.com/
  - Hollow Knight: Silksong 官方站：https://www.hollowknightsilksong.com/
  - Godot 官方站：https://godotengine.org/

## 当前资产边界

- 主角：`res://assets/sprites/player/generated_hero_v2/manifest.json`
  - 用于玩家可控角色，包含 idle、run、jump、fall、land、dash、attack、hook、hurt、death、heal 等关键帧映射。
  - 在 `res://scenes/Player.tscn` 中缩放到 `Vector2(0.66, 0.66)`，保证 192px 帧在平台上可读。
- 敌人：沿用开放 GothicVania 敌人视觉，不复用主角或 NPC 家族素材。
- NPC：可保留同族/同画风友方识别，使用 talk badge / halo 与敌人区分。
- 地图和平台：使用 Mossy Cavern 纹理类素材，地图叠加房间块、当前房间描边、玩家指示条和图例。

## 手感与动作规格

- 移动速度：`285`
- 跳跃速度：`-710`
- 重力：`1850`
- 冲刺速度：`780`
- 土狼时间：`0.12s`
- 跳跃缓存：`0.12s`
- 短跳削减：`0.48`
- 冲刺时长：`0.16s`
- 冲刺冷却：`0.60s`
- 攻击前冲：`92`

## 核心玩法闭环

- 第1章苔钟庭：3 把钟钥 -> 后门捷径 -> Boss 门 -> 锈冠守卫 -> 王冠裂片。
- 章节链路：CH01-CH06 均可通过数据驱动注入到主场景验证。
- 地图系统：集成 MetSys autoload 与本地地图 overlay，记录已探索房间、当前房间、收集物和交互对象状态。
- 战斗系统：左键普通攻击、右键消耗能量治疗、E 技能重击、F 交互；敌人以近战读招和身体范围攻击为主。

## 验收标准

- Godot 工程能用 `--path godot` 正常加载。
- 主角运行时必须加载 `generated_hero_v2` manifest。
- 背包和地图显示中文道具/房间/路线文本。
- 敌人视觉不能复用主角素材。
- CH01-CH06 数据闭环、战斗、音画、地图可读性和 MetSys 验证均通过。
