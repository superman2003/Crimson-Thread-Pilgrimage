# 空洞骑士与丝之歌对标优化规格

## 0. 引擎与当前事实

- 当前可运行版本是浏览器 Canvas 游戏，入口是 `index.html`、`src/main.js`、`server.js`，访问地址是本地 Node 静态服务。
- 仓库里存在 `godot/` 目录和 Godot 工程文件，但当前 5173 页面没有使用 Godot 引擎运行；`godot/assets/...` 主要作为本地素材目录。
- 本轮不迁移引擎，先优化当前 Web Demo 的手感、战斗反馈、地图目标与 QA。

## 1. 对标边界

只对标质量目标，不复制商业 IP。

- 允许借鉴：紧凑平台动作手感、互联探索结构、能力门、捷径、清晰读招、命中反馈、低遮挡 HUD、浏览器试玩 QA 标准。
- 禁止复制：Hollow Knight / Silksong 的角色名、商标词、角色轮廓、地图命名、Boss 招式、美术素材、音乐旋律、UI 图标。
- 资料口径：
  - Hollow Knight 官方站：https://www.hollowknight.com/
  - Silksong 官方站：https://hollowknightsilksong.com/
  - Team Cherry Silksong 发布页：https://www.teamcherry.com.au/blog/hollow-knight-silksong-revealed

## 2. 核心体验目标

- 2D 侧视动作平台，重点是“移动、跳跃、冲刺、近战、探索、返场”。
- 玩家 30 秒内能进入战斗，2 分钟内能看见钥匙/捷径/Boss 门的目标闭环。
- 每个章节遵循：安全教学 -> 小考平台 -> 压力战斗 -> 安静回收 -> Boss 前捷径 -> Boss -> 出口。
- 视觉上主角、敌人、背景必须分层清楚；主角使用原创 image2 关键帧，敌人使用开放/免费素材，不混用同族外观。

## 3. P0 立即落地

1. 手感容错
   - 新增跳跃缓冲 `jumpBuffer`。
   - 土狼时间从 `0.09s` 提升到 `0.12s`。
   - 短按跳高控制：松开跳跃后削减上升速度。
   - 落地反馈：`landTimer`、落地粒子和动画切换。

2. 战斗反馈
   - 近战增加轻微前冲。
   - 命中增加 `hitStop`，普通敌短停顿，Boss 稍长停顿。
   - 普通敌命中后有轻微击退和白闪。
   - 攻击弧光更明显，避免只出现淡矩形。

3. 敌人读招
   - `charger` 冲刺前增加 `windup`。
   - `shooter` 发射前增加 `alertTimer` 和预警线。
   - 敌人绘制时显示预警，不和背景混色。

4. 中文与 IP 安全
   - 修复 `src/gameData.js` 章节中文乱码。
   - 测试增加乱码哨兵和 IP 禁词哨兵。

## 4. P1 本轮同步落地

1. 钥匙和捷径
   - 每章保留 3 个钥匙点，玩家碰到后真实收集。
   - 收集 3 把钥匙后靠近拉杆打开 Boss 前捷径。
   - HUD 显示钥匙、捷径、Boss 门状态。

2. 地图引导
   - Canvas 内增加方向箭头。
   - Boss 未死时指向 Boss / 目标；Boss 已死时指向出口。
   - QA hook 暴露 `keys`、`shortcutOpen`、`lastEvent`、`hitStop`、`playerX`。

3. 低遮挡 HUD
   - 保持 DOM HUD 简洁，不能覆盖中心战斗区域。
   - 移动端触控按钮保持底部，不遮挡主角附近区域。

## 5. 验收标准

- `npm test` 必须通过。
- `npm run test:browser` 必须通过并输出桌面/手机截图。
- 浏览器 QA 必须断言：
  - 主角 32 帧 image2 素材全部加载。
  - 主角源帧为 `192x192`，视觉渲染为 `128x128`，碰撞盒仍为 `34x54`。
  - 初始 `keys=0`、`shortcutOpen=false`、`hitStop=0`。
  - 页面文案无明显中文乱码。
- 本地服务必须返回 200。

## 6. 回滚

- 手感参数和战斗反馈集中在 `src/main.js`，可按函数回滚。
- 章节中文和节奏数据集中在 `src/gameData.js`。
- 测试只增加质量哨兵，不改变启动入口。
