# 丝之歌式类银河城地图设计研究

时间：2026-06-19 14:58:27

## 参考边界

本项目只参考类银河城通用设计语言，不复刻《Hollow Knight: Silksong》的具体地图、房间布局、路线、命名或商业美术。

可借鉴的是设计原则：

- 清晰的大目标方向，例如持续向上、向核心、向圣城或王座推进。
- 早期相对清晰，中后期逐步开放分支、回流和破序。
- 每个区域有强主题、强地标和可记忆的路径形状。
- 让玩家先看到无法抵达的路线，再通过能力或捷径回到那里。
- Boss 前保留紧张感，但死亡回跑必须短。
- 支线不是纯奖励房，而是补充世界观、NPC、资源和路线理解。

## 地图结构规则

| 规则 | 生成器落地 |
| --- | --- |
| 区域要有宏观形状 | `entry_spine`、`tall_shaft`、`loop_gallery`、`reward_cave`、`wide_canal`、`boss_run` |
| 房间要有设计职责 | `archetype` 字段 |
| 每个房间要说明为什么存在 | `design_goal` 字段 |
| 每个房间要说明主要移动玩法 | `traversal_focus` 字段 |
| 每个房间要说明奖励或回报 | `reward_contract` 字段 |
| 节奏要能验证 | `pacing_role` 字段 |
| 捷径必须有价值 | `shortcut_value.distance_saved >= 1` |
| 能力门不能软锁 | 验证器按能力获取顺序模拟可达性 |
| Boss 前短跑 | 存档点到 Boss 不超过 3 步 |

## 房间 Archetype

| Archetype | 职责 |
| --- | --- |
| `entry_gate` | 安全入口、教学、第一地标 |
| `region_hub` | 区域内多出口枢纽，后续分支回流 |
| `main_spine` | 主线推进骨架 |
| `vertical_shaft` | 纵向记忆点和后期能力压缩路径 |
| `loop_gallery` | 绕行后回到熟悉房间 |
| `reward_pocket` | 可选风险奖励袋 |
| `ability_tutorial` | 能力获得后的即时教学 |
| `shortcut_lock` | 先看到锁住捷径，再从背面打开 |
| `hidden_reward` | 隐藏奖励，不阻塞主线 |
| `overlook_branch` | 先看见目标，后面再抵达 |
| `return_passage` | 下层回流通道 |
| `preboss_run` | Boss 前短跑道 |
| `boss_arena` | Boss 战场 |

## 当前实现方向

本轮把研究结果直接接进生成器：

- `tools/build_silksong_scale_map_blueprints.py`：生成房间设计契约。
- `tools/blueprint_to_godot_runtime.py`：把契约转入 Godot runtime JSON，并用 archetype 影响平台模板。
- `tools/validate_generated_godot_runtime.py`：验证每章必须覆盖关键 archetype。
