# 类丝之歌开源仓库学习调研

时间：2026-06-15 20:20:03

## 结论

当前项目还只能算竖切 Demo。要往类丝之歌方向走，建议继续以 `KoBeWi/Metroidvania-System` 做地图主线，吸收其他开源仓库的技能、背包、房间组织、存档与关卡拓扑思路。不要复制 Hollow Knight/Silksong 的角色、美术、命名和具体设定。

## 已拉取到本地的参考仓库

- `artifacts/source_repos/Metroidvania-System`：MIT。已接入项目，继续作为地图、探索、收集物持久化核心。
- `artifacts/source_repos/godot-platformer-2d`：MIT。GDQuest 类银河城平台动作和关卡教学结构。
- `artifacts/source_repos/toziuha-night-oota`：MIT 代码。可学习完整类恶魔城项目结构；不要复制图片、音乐、精灵等资产。
- `artifacts/source_repos/godot-gamelevels-blueprint`：MIT。可学习房间连接蓝图和关卡拓扑编辑。
- `artifacts/source_repos/bread-adventure`：Apache-2.0。Godot 4.5，含背包、地图 UI、传送、隐藏墙、存档点等结构。
- `artifacts/source_repos/metroidvania-godot-4`：MIT。含 SaveManager、WorldStash、门、存档点、基础敌人和 powerup。

## 额外建议学习但暂不接入

- `OctoD/godot-gameplay-abilities`：MIT，能力容器/技能授予架构很好；但为 GDExtension，先学架构，不直接加重依赖。
- `expressobits/inventory-system`：MIT，Godot 4 背包系统成熟；当前项目先做轻量技能/道具数据表，后续背包复杂后再评估接入。
- `Noah-Erz/Ultimate-Platformer-Controller-2D`：MIT，平台手感参数很完整；不要替换现有玩家脚本，逐步吸收土狼时间、跳跃缓冲、墙跳、下劈等状态。

## 学习映射

- 大地图：Metroidvania-System + gamelevels-blueprint。
- 技能解锁：godot-gameplay-abilities 的容器思想 + 本项目 `progression_plan.json`。
- 道具/护符：expressobits/inventory-system 的资源化思路 + Bread Adventure 的 UI 分层。
- 存档/进度：Metroidvania-System SaveData + metroidvania-godot-4 SaveManager。
- 房间切换：MetSys RoomInstance + Portal/Elevator 示例。
- 平台动作：GDQuest 和 Ultimate Platformer Controller 的手感参数。
