# 绯线远征实施计划

## 目标
- 交付一个原创 2D 动作平台独立游戏，主题可类比高速横版探索动作，但不使用任何外部版权素材。
- 至少 6 章，每章场景风格、怪物、Boss、危险机制不同。
- 完成本地可运行、可测试、可交付的最小闭环。

## 技术栈决策
- 使用原生 HTML5 Canvas + JavaScript ES Modules。
- 使用 Node.js 内置 HTTP 服务做本地运行。
- 素材通过 Canvas 程序化生成，音效通过 WebAudio 程序化生成。
- 不引入外部图片、音频、字体或运行依赖。

## 任务拆分
- Agent-0：集成实现、冲突控制、最终验证。
- Agent-1：只读定位项目现状与最短路径。
- Agent-Blueprint：章节、敌人、Boss、资产生成方案。
- Agent-3：由 Agent-0 本地接管前端 Canvas 实现。
- Agent-4：测试清单与交付验证。

## 验收标准
- `npm test` 通过。
- `node server.js` 可启动本地服务。
- 浏览器可进入游戏，完成移动、跳跃、冲刺、攻击、Boss 战、章节推进。
- 游戏数据包含 6 章，每章不同怪物与 Boss。
- 静态检查确认没有外部网络素材引用。
