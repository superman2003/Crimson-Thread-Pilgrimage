# 大地图技能道具系统扩展计划

时间：2026-06-15 20:20:03

## 当前判断

当前版本是可玩的竖切 Demo：7 个房间、1 个章节、5 个普通敌人、1 个 Boss、基础背包和地图。下一步目标是做成 6 章、约 120+ 房间、能力门驱动的大地图。

## 里程碑

1. 数据契约
   - 使用 `godot/data/progression_plan.json` 作为技能、道具、护符、门禁类型的第一版契约。
   - 所有房间配置增加 `required_abilities`、`locks`、`rewards`、`connections`。

2. 技能系统
   - 在 Godot 新增 `progression_state.gd` 管理已获得技能、道具、护符槽、章节钥匙。
   - 玩家动作脚本从 `progression_state` 判断冲刺、钩索、二段跳、墙跳、下劈等是否可用。
   - 第一批实现：绯线冲刺、回声钩索、双线跃。

3. 大地图结构
   - 把当前单场景桥接逐步拆为 MetSys 原生 `RoomInstance`。
   - 每章独立 MapData 和房间场景，先做第一章 18 个房间。
   - 保留 MetSys 存储对象和探索记录。

4. 道具与护符
   - 背包从纯文字扩展为分类列表：钥匙、能力、护符、地图、材料。
   - 护符使用 `cost/slot/effect`，先实现 5 个轻量效果。

5. 存档与检查点
   - 保存：技能、道具、生命/能量上限、已探索房间、已收集物、Boss 状态。
   - 检查点：灯/钟台，支持回血、保存、设置复活点。

6. 验证
   - 新增门禁验证：没有能力不能进，有能力可进。
   - 新增技能验证：技能解锁后输入生效，未解锁时不生效。
   - 新增地图规模验证：房间数、连接数、死路、回环、捷径数量。

## 第一阶段可执行任务

- 新增 `progression_state.gd`。
- 让 `player_controller.gd` 消费技能解锁状态。
- 把当前 7 房间扩到第一章 18 房间。
- 把 `map_rooms[]` 加 `connections` 和 `required_abilities`。
- 实现 3 个能力门：冲刺缺口、钩索铃环、二段跳高度门。

## 风险

- 直接替换玩家控制器会破坏手感，必须渐进式加能力。
- 直接引入完整背包插件可能过重，先做轻量数据表。
- 大地图如果继续单场景堆配置会失控，必须逐步切 RoomInstance。
