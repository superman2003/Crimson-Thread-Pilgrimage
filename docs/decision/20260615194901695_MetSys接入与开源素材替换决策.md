# MetSys接入与开源素材替换决策

时间：2026-06-15 19:49:01

## 结论

本轮将 Godot Demo 的运行时素材切换到 OpenGameArt 的 Ansimuz GothicVania Cemetery Pack，并将 KoBeWi/Metroidvania-System 作为 Godot 插件接入工程。当前采用桥接式接入：现有单场景程序化关卡继续运行，MetSys 负责地图数据、探索记录、可存储对象记录和保存数据结构。

## 采用内容

- `godot/addons/MetroidvaniaSystem`：从 `KoBeWi/Metroidvania-System` 引入，授权 MIT。
- `godot/MetSysSettings.tres`、`godot/MapData.txt`：配置 MetSys 设置和 7 个 Demo 房间 cell。
- `godot/scripts/metroidvania_bridge.gd`：桥接现有关卡房间、玩家探索、收集物、机关和 Boss 状态到 MetSys。
- `godot/assets/sprites/gothicvania`：新运行时素材树，避免 Godot import 缓存继续使用旧素材。

## 素材授权

- Ansimuz GothicVania Cemetery Pack：Public domain，允许个人/商业使用；本轮使用 PNG 图像素材，未使用包内音乐。
- Kenney Platformer Art Deluxe：CC0；仅作为钥匙和机关小件补充。
- KoBeWi/Metroidvania-System：MIT。

## 保留取舍

当前没有把每个房间拆成 MetSys 原生 `RoomInstance` 场景，因为本 Demo 是单场景动态构建地图。若下一阶段要做完整 6 章规模，建议把章节和房间拆成原生 Godot 场景，并用 MetSys 的 `RoomInstance`、Map Editor、scene assignment 接管房间切换。

## 风险

- 当前桥接层能记录探索和收集，但不是 KoBeWi 示例项目那种完全原生 RoomInstance 工作流。
- 旧素材文件仍可作为归档存在，但运行时代码已切到 `res://assets/sprites/gothicvania/...`。
