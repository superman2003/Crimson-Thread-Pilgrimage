# 不可再分发素材包清单与开源边界

时间：2026-06-16 16:20:14.251

## 结论

当前项目本地目录里确实存在“可用于游戏项目，但不适合直接随公开 GitHub 仓库再分发”的素材包。  
只把限制写进 MD 不足以解决授权问题：如果把素材文件本身提交到公开仓库，公开仓库就是一次素材再分发。

开源策略建议：

- 根仓库提交代码、Godot 工程、可再分发素材、文档、工具脚本。
- 对再分发受限素材：要么替换为 CC0 / Public Domain / MIT / Apache-2.0 / CC-BY 等可再分发素材，要么从 Git 提交中排除，并在 README 写“用户自行下载并放到指定路径”。
- `artifacts/source_repos`、`artifacts/source_assets`、`artifacts/third_party` 只作为本地调研/下载缓存，不提交。

## README 需要明确写的边界

1. 本仓库不是素材包仓库，第三方素材只用于演示游戏项目。
2. 再分发受限素材不会作为公开仓库内容发布；若某分支需要它们，用户需自行从原作者页面下载并遵守原授权。
3. 任何 fork / release 不应把 `artifacts/source_*`、`artifacts/third_party`、`artifacts/debug` 作为发布包上传。
4. CC-BY 素材必须保留署名信息，不能只写“免费素材”。
