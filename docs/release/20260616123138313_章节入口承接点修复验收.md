# 章节入口承接点修复验收

时间：2026-06-16 12:31:38.313

## 问题

第一章打完 Boss 后走到章节出口，虽然 runtime 已经加载第二章，但第二章默认 `player_start` 仍是 `[120, 500]`，与第一章开局坐标一致，且缺少明确章节进入提示，玩家体感像被传送回第一章起点。

## 修复

- `MainLevel` 新增切章入口覆盖状态：
  - `entry_spawn_override_enabled`
  - `entry_spawn_override`
  - `entered_from_config_id`
  - `transition_entry_message`
- `_transition_to_next_chapter()` 读取下一章 `entry_from_previous`，把玩家放到章节承接点，而不是默认开局点。
- CH02-CH06 新增 `entry_from_previous`：
  - `position: [520, 500]`
  - `from_runtime_config_id`
  - 章节进入提示文案
- `_show_chapter_entry_message()` 在切章后显示明确章节标题和承接说明。
- `all_chapters_demo_validation.gd` 新增断言：
  - 下一章使用 transition entry spawn。
  - 下一章记录上一章 runtime id。
  - 下一章出生点不是默认起点。
  - 下一章显示 entry message。
