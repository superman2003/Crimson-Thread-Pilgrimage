# MetSys墙路地图桥接验收

时间：2026-06-16 13:47:44

## 结果

已将 KoBeWi Metroidvania-System 用作 Godot 试玩版的地图拓扑数据层。现有程序化关卡继续负责平台、碰撞、Boss门和章节衔接；`metroidvania_bridge.gd` 负责把章节 `map_rooms` 转成 MetSys cell，并自动判定墙、路、锁门、章节出口。

## 改动

- `godot/scripts/metroidvania_bridge.gd`
  - 每个房间创建 MetSys cell，默认四边为墙：`0`。
  - 按 `map_rooms` 顺序打开主路线通道：`1`。
  - 遇到上下层房间时自动生成 connector cell，避免斜向房间在 MetSys 图里断开。
  - 记录 boss gate 锁门边、chapter exit 出口边、墙/通道/连接 cell 统计。
- `godot/scripts/main_level.gd`
  - 地图 overlay 的路线线段改成转角连线。
  - 房间边缘增加墙体视觉挡板。
  - boss gate 用琥珀色标记，chapter exit 用蓝色标记。
  - 地图文本显示 MetSys 的 walls / passages / locks / connectors 统计。
- `godot/scripts/metsys_validation.gd`
  - 新增墙、通道、主路线、锁门边、connector cell 验证。

## 验证

```text
METSYS_VALIDATION_PASS cells=20 walls=41 passages=39 routes=12 locked=1 connectors=8 discovered=2 registered=5 stored=1
INVENTORY_MAP_VALIDATION_PASS backpack=true map=true visited=2 currency=38
ALL_CHAPTERS_DEMO_VALIDATION_PASS chapters=6 complete=true
DEMO_VALIDATION_PASS enemies=9 hazards=9 keys=0 boss=1 combat_clear=true complete=true
COMBAT_VALIDATION_PASS player_hits=true hook_hits=true skill_E=true heal_RMB=true interact_F=true ai_profiles=5 enemy_ai_damage=true
MAP_READABILITY_VALIDATION_PASS rooms=12 panel=1176.0x640.0 platform_children=16
```

## 决策

- 采用 MetSys 的语义：`0=墙`，`1=通道/出口`，`-1=同房间内部无边界`。
- boss gate 在地图层作为“已知锁门边”高亮，实际能否进入仍由清怪、Boss解锁和交互逻辑控制。
- 章节出口作为高辨识蓝色通道边，避免玩家打完 Boss 后误以为回到起点。

## 风险

- 当前仍是单场景程序化关卡，MetSys 负责拓扑、探索、持久化对象和地图表达；如果后续要完全用 MetSys 原生房间切换，需要再把每个房间拆成 `RoomInstance` 场景。
- Godot headless 退出时项目已有 ObjectDB 泄漏警告，本轮验证脚本 exit 0，未作为阻塞项。
