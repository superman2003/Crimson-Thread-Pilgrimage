# Boss后章节衔接修复验收

时间：2026-06-16 12:24:20.778

## 问题

第一章打完 Boss 后无法进入下一章。原因不是章节路径缺失，而是 `boss_back_wall` 作为实体碰撞墙仍堵在 Boss 房间和章节出口之间；旧验证只检查“出口节点存在”，没有实际触发章节切换，因此漏掉了真实流程卡死。

## 修复

- Boss 死亡后调用 `_open_post_boss_route()`，移除 `boss_back_wall`、`post_boss_wall`、`chapter_exit_wall` 这类后 Boss 封路墙。
- 增加通道打开特效 `PostBossRouteOpenRing`，让墙消失时有视觉反馈。
- 章节出口增加防重复切换状态 `chapter_transitioning`。
- 章节出口支持两种进入方式：
  - 靠近已开启出口时自动进入下一章。
  - 靠近出口按 F 手动进入下一章。
- `get_demo_state()` 暴露 `post_boss_route_open`，供回归脚本验证。
