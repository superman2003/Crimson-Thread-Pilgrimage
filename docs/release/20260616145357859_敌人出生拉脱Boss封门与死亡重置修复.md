# 敌人出生、拉脱、Boss封门与死亡重置修复

## 结果

已修复普通怪刷进墙体/平台的问题，新增普通怪小范围拉脱，Boss 开战后临时封门，玩家死亡后普通怪和 Boss 状态重置。

## 改动

- `godot/scripts/enemy_actor.gd`
  - 新增 `leash_radius`、出生点记录、最大血量记录。
  - 普通怪离开出生范围会回巢，攻击/冲刺/击退都会被限制在拉脱半径内。
  - 新增 `reset_for_respawn()`，用于后续复活规则扩展。

- `godot/scripts/main_level.gd`
  - 敌人出生点按平台顶面吸附，并根据素材裁剪高度检查是否和墙、门、上方平台重叠。
  - Boss 门触发后生成临时 `boss_arena_lock_wall`，Boss 死亡或玩家死亡后移除。
  - 监听 `player.player_respawned`，死亡后清掉场上敌人和 Boss，下一帧按关卡配置重新刷怪。
  - `get_demo_state()` 暴露 `spawn_clear`、`leash_radius`、`boss_arena_locked`，便于回归验证。

- `godot/scripts/ch02_demo_validation.gd`
  - 增加 CH02 敌人出生不撞墙、普通怪拉脱、Boss 封门、死亡重置、Boss 重启验证。

- `godot/scripts/all_chapters_demo_validation.gd`
  - 全章节增加敌人出生清洁、普通怪拉脱、Boss 房封门/击杀解锁验证。

## 风险

- 普通怪默认拉脱半径为 240 像素，后续如果某个房间想做更宽的精英怪巡逻，需要在敌人配置里单独加 `leash_radius`。
- 死亡后普通怪会按配置重刷，Boss 也会回到未开战状态；Boss 已击杀后的章节完成状态不重置。
